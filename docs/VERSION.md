# GitHub Guide Tool - 版本规划

## 版本号规范

采用语义化版本 (Semantic Versioning)：`MAJOR.MINOR.PATCH`

- **MAJOR**: 重大变更，可能包含不兼容的 API 修改
- **MINOR**: 新功能，向后兼容
- **PATCH**: Bug 修复，向后兼容

## 版本发布流程

### 1. 开发阶段
- 功能开发和测试
- 文档更新
- 版本号更新

### 2. 测试阶段
- 功能测试
- 兼容性测试（Chrome/Edge）
- 性能测试
- 安全审计

### 3. 发布准备
- CHANGELOG 更新
- README 更新
- 打包构建
- 创建 GitHub Release

### 4. 发布
- 发布到 GitHub Releases
- 更新文档站点（如有）
- 发布公告

### 5. 后续支持
- Bug 修复（Patch 版本）
- 用户反馈收集
- 下一版本规划
